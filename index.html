HTML — HyperText Markup Language

What does HTML stand for?

HTML = HyperText Markup Language

Let's break that down:

  • HyperText — Text that links to other text. When you click a link on a webpage and it takes you to another page — that's hypertext. The "hyper" means "beyond" — it goes beyond normal text by connecting pages together.
  • Markup — You "mark up" content with tags to tell the browser how to display it. Like how a teacher marks up your essay with red pen, you mark up text with tags like <h1>, <p>, <img>.
  • Language — It's a language that browsers understand. But it's NOT a programming language — it can't do logic, loops, or calculations. It's a markup language for structuring content.
✅ Key fact for exams
HTML is NOT a programming language. It's a markup language. If the exam asks "Is HTML a programming language?" — the answer is No.

How the Web Works (30-second version)

  1. You type a URL in your browser (like google.com)
  2. Browser sends a request to a server
  3. Server sends back an HTML file
  4. Browser reads the HTML and renders (displays) the page
  5. CSS makes it look good, JavaScript makes it interactive

HTML = Structure (skeleton) · CSS = Style (clothes) · JavaScript = Behavior (muscles)

HTML Document Structure

Every HTML page follows this skeleton. This is the foundation of everything:

<!DOCTYPE html>        <!-- Declaration: tells browser this is HTML5 -->
<html lang="en">        <!-- Root element. lang="en" = English -->
<head>                  <!-- Metadata — NOT visible on page -->
  <meta charset="UTF-8">              <!-- Character encoding -->
  <meta name="viewport"
    content="width=device-width, initial-scale=1.0"> <!-- Mobile responsive -->
  <title>My Page</title>             <!-- Shows in browser tab -->
  <link rel="stylesheet" href="style.css"> <!-- Link to CSS -->
</head>
<body>                  <!-- ALL visible content goes here -->
  <h1>Hello World</h1>
  <p>This is a paragraph.</p>
  <script src="app.js"></script>     <!-- JS at bottom of body -->
</body>
</html>
💡 Exam tip
<!DOCTYPE html> is NOT a tag — it's a declaration. It tells the browser to use HTML5 mode. This gets asked.

CSS — Cascading Style Sheets

What does CSS stand for?

CSS = Cascading Style Sheets

Let's break that down:

  • Cascading — When multiple rules target the same element, CSS "cascades" — it follows a specific priority order to decide which rule wins. Think of water cascading down: it follows the path of least resistance (highest priority).
  • Style — CSS controls the visual appearance: colors, fonts, spacing, layout, animations.
  • Sheets — Styles are written in files (stylesheets) that the browser reads and applies to HTML.
✅ Key fact for exams
CSS is NOT a programming language. It's a style sheet language. HTML = Structure. CSS = Style. JavaScript = Behavior. This trinity gets asked frequently.

Three Ways to Add CSS (Very Frequently Asked)

1. Inline CSS

Written directly inside the HTML element using the style attribute. Highest priority.

<p style="color: red; font-size: 18px;">This text is red.</p>
<h1 style="background-color: yellow;">Yellow background</h1>

2. Internal CSS

Written inside a <style> tag in the <head> section. Applies to that one HTML file only.

<head>
  <style>
    p {
      color: blue;
      font-size: 16px;
    }
    h1 {
      background-color: yellow;
    }
  </style>
</head>

3. External CSS

Written in a separate .css file and linked to HTML using <link>. Best practice — reusable across many pages.

<!-- In the HTML file (inside <head>) -->
<link rel="stylesheet" href="style.css">

/* In the style.css file */
p {
  color: blue;
  font-size: 16px;
}
TypeWhere it's writtenScopePriority
InlineInside the HTML tag's style attributeThat element onlyHighest
Internal<style> in <head>That HTML page onlyMiddle
ExternalSeparate .css fileAll linked pagesLowest (but wins over internal when equal specificity)
💡 Exam tip
Precedence order: Inline > Internal > External. But inside the same stylesheet, the rule that appears last wins (that's the "cascading" part). If specificity is equal, last rule wins.

CSS Selectors

Selectors target which HTML elements a CSS rule applies to.

/* Element selector — targets all <p> tags */
p { color: red; }

/* Class selector — targets elements with class="highlight" */
.highlight { background: yellow; }

/* ID selector — targets element with id="main-title" */
#main-title { font-size: 32px; }

/* Universal selector — targets EVERY element */
* { box-sizing: border-box; }

/* Descendant selector — any <p> inside a <div> (any depth) */
div p { color: blue; }

/* Child selector — only DIRECT children */
div > p { color: green; }

/* Adjacent sibling — <p> immediately AFTER an <h2> */
h2 + p { margin-top: 0; }

/* Attribute selector — targets <input type="text"> */
input[type="text"] { border: 1px solid #ccc; }

The CSS Box Model

Every HTML element is a rectangular box. The box model describes the layers around that box.

/*
  Box Model — from inside out:

  +------------------------------------------+
  |              MARGIN (outside)             |
  |   +----------------------------------+   |
  |   |         BORDER                   |   |
  |   |   +--------------------------+   |   |
  |   |   |       PADDING            |   |   |
  |   |   |   +------------------+   |   |   |
  |   |   |   |    CONTENT       |   |   |   |

div {
  width: 300px;         /* content width */
  padding: 20px;        /* space INSIDE the border */

The position Property

/* static — default. top/left have NO effect */

/* relative — stays in flow, offset from original spot */
div { position: relative; top: 10px; left: 20px; }

/* absolute — removed from flow, relative to nearest positioned ancestor */
.parent { position: relative; }
.child  { position: absolute; top: 0; right: 0; }

/* fixed — relative to viewport (stays on scroll) */
.navbar { position: fixed; top: 0; width: 100%; }

/* sticky — normal until scrolled past, then sticks */

z-index

Controls stacking order. Higher z-index = on top.

.modal { position: fixed; z-index: 100; }
.navbar { position: fixed; z-index: 200; }
/* Navbar appears ON TOP of modal */

p { z-index: 999; } /* IGNORED — p is static by default */
✅ Key fact for exams
z-index only works on positioned elements (relative, absolute, fixed, sticky). Does nothing on static.

5. Type Coercion — The #1 Tricky Topic

JavaScript automatically converts values from one type to another. This is called implicit type coercion and it causes many surprising results.

🔑 Key Rule
The + operator: if either operand is a string, it concatenates. All other arithmetic operators (-, *, /, %) convert strings to numbers first. 
// + with strings = concatenation
"1" + 2        // "12"  — number 2 becomes string "2"
1 + "2"        // "12"  — same result
"5" + 3 + 2   // "532" — left to right: "5"+3="53", "53"+2="532"
5 + 3 + "2"   // "82"  — 5+3=8 first (numbers), then 8+"2"="82"

// - * / always convert to number
"5" - 3        // 2   — "5" becomes number 5
"6" * "2"      // 12  — both become numbers
"10" / "2"    // 5
"abc" - 1     // NaN — "abc" can't be a number

// Boolean coercion
true + 1       // 2  — true becomes 1
false + 1      // 1  — false becomes 0
true + true    // 2

// null and undefined
null + 1       // 1  — null becomes 0
undefined + 1  // NaN

// Floating point trap
0.1 + 0.2       // 0.30000000000000004 — NOT 0.3!
0.1 + 0.2 == 0.3 // false — floating point precision issue
📋 Exam Pattern
Output prediction questions on type coercion appear in nearly every JS exam. Memorise: "1" + 2 = "12" and "5" - 3 = 2. The key: + concatenates strings, everything else forces numeric conversion.

13. Closures

A closure is when an inner function remembers variables from its outer function, even after the outer function has finished executing.

function makeCounter() {
  let count = 0;          // this variable lives in makeCounter's scope

  return function() {     // inner function — this is the closure
    count++;
    return count;
  };
}

const counter = makeCounter(); // makeCounter finished, but count survives
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3

// Each call to makeCounter() creates a SEPARATE closure
const counter2 = makeCounter();
console.log(counter2()); // 1 — independent from counter
💡 Exam tip
Closures are used for data privacy — variables inside a closure can't be accessed from outside. The inner function "closes over" the outer variables.

15. Hoisting

🔑 Key Fact — Hoisting Summary
  • var declarations → hoisted to top of function, initialised as undefined
  • var assignments → NOT hoisted, stay where they are
  • function declarations → fully hoisted (name + body), callable anywhere
  • let / const → hoisted but in Temporal Dead Zone (TDZ) — throws ReferenceError if accessed before declaration
  • function expressions / arrow functions → follow the rules of the variable they're assigned to
// var hoisting
console.log(x); // undefined — declaration hoisted, assignment not
var x = 10;
console.log(x); // 10

// Function declaration hoisting — fully hoisted
sayHello(); // "Hello!" — works even before the function definition
function sayHello() { console.log("Hello!"); }

// let TDZ
console.log(y); // ReferenceError: Cannot access 'y' before initialization
let y = 5;

// Function expression — NOT hoisted
greet(); // TypeError: greet is not a function
var greet = function() { console.log("Hi"); };

16. Truthy & Falsy Values

In JavaScript, every value is either truthy or falsy when evaluated in a boolean context (like an if statement).

⚠️ The 6 Falsy Values — Memorise These
false  |  0  |  "" (empty string)  |  null  |  undefined  |  NaN

Everything else is truthy — including "0", "false", [], {}. 
// Falsy — all these if-blocks do NOT run
if (0)         { /* skipped */ }
if ("")        { /* skipped */ }
if (null)      { /* skipped */ }
if (undefined) { /* skipped */ }
if (NaN)       { /* skipped */ }

// Truthy — these DO run
if ("0")        { /* runs — non-empty string, even if "0" */ }
if ([])         { /* runs — empty array is truthy! */ }
if ({})         { /* runs — empty object is truthy! */ }
if ("false")    { /* runs — non-empty string is truthy */ }
if (-1)         { /* runs — any non-zero number is truthy */ }

17. NaN — Not a Number

typeof NaN         // "number" — NaN is paradoxically of type number
isNaN(NaN)        // true
isNaN("abc")      // true  — "abc" gets coerced to NaN
isNaN("123")      // false — "123" coerces to number 123
isNaN(undefined) // true  — undefined is NaN when converted

NaN === NaN       // false — NaN is NOT equal to itself!

Number.isNaN(NaN)   // true  — stricter, doesn't coerce
Number.isNaN("abc") // false — "abc" is not NaN, it's a string

9. GROUP BY + HAVING

GROUP BY collapses rows that share a value into one group so aggregate functions work per group.

-- Count employees per department
SELECT dept, COUNT(*) AS headcount
FROM   employees
GROUP BY dept;
-- IT→3, HR→2, Finance→1

-- Average salary per department
SELECT dept, AVG(salary) AS avg_sal
FROM   employees
GROUP BY dept
ORDER BY avg_sal DESC;
-- IT: 85000, Finance: 80000, HR: 60000

HAVING — filtering groups (WHERE for GROUP BY)

-- Departments where average salary exceeds 70000
SELECT dept, AVG(salary) AS avg_sal
FROM   employees
GROUP BY dept
HAVING AVG(salary) > 70000;
-- Returns: IT (85000), Finance (80000)

-- Departments with more than 1 employee, earning over 65k avg
SELECT dept, COUNT(*) AS headcount, AVG(salary) AS avg_sal
FROM   employees
WHERE  salary > 50000          -- WHERE filters ROWS before grouping
GROUP BY dept
HAVING COUNT(*) > 1           -- HAVING filters GROUPS after grouping
ORDER BY avg_sal DESC;
💡 Exam tip
WHERE vs HAVING — the #1 tested difference:
WHERE filters individual rows — runs BEFORE grouping. Cannot use aggregate functions.
HAVING filters groups — runs AFTER GROUP BY. Can use aggregate functions like COUNT(), AVG().
📋 Exam info
"Which clause is used to filter groups?" → HAVING
"Can you use WHERE with aggregate functions?" → No — use HAVING instead.

10. JOINs

JOINs combine rows from two or more tables based on a related column.

For JOIN examples, we use a second table:

CREATE TABLE departments (
    dept_name VARCHAR(30) PRIMARY KEY,
    location  VARCHAR(50)
);

-- Data: IT→Mumbai, HR→Delhi, Finance→Pune, Marketing→Bangalore
-- Note: 'Marketing' has no employees. Frank/Alice/Carol are in 'IT'.

INNER JOIN — only matching rows from both tables

SELECT e.name, e.dept, d.location
FROM   employees e
INNER JOIN departments d ON e.dept = d.dept_name;
-- Returns: All employees matched to a dept. Marketing (no employees) excluded.
-- Alice→Mumbai, Bob→Delhi, Carol→Mumbai, Dave→Pune, Eve→Delhi, Frank→Mumbai

LEFT JOIN — all rows from left table + matches from right

SELECT e.name, d.location
FROM   employees e
LEFT JOIN departments d ON e.dept = d.dept_name;
-- Returns: ALL employees. If no dept match, location = NULL.
-- Same result here because all employees have valid depts.

RIGHT JOIN — all rows from right table + matches from left

SELECT e.name, d.dept_name
FROM   employees e
RIGHT JOIN departments d ON e.dept = d.dept_name;
-- Returns: ALL departments. Marketing appears with name = NULL (no employee).

FULL OUTER JOIN — all rows from both tables

SELECT e.name, d.dept_name
FROM   employees e
FULL OUTER JOIN departments d ON e.dept = d.dept_name;
-- Returns: Everything. NULLs where no match exists on either side.
-- Note: MySQL doesn't support FULL OUTER JOIN directly.
-- Workaround: LEFT JOIN UNION RIGHT JOIN

Self Join — joining a table to itself

-- Show each employee with their manager's name
SELECT
    e.name        AS employee,
    m.name        AS manager
FROM   employees e
LEFT JOIN employees m ON e.manager_id = m.emp_id;
-- Alice → NULL (no manager), Bob → Alice, Carol → Alice, etc.
🔑 Key fact
JOIN visual summary:
INNER JOIN = intersection only
LEFT JOIN = all left + matching right (NULLs for unmatched right)
RIGHT JOIN = all right + matching left (NULLs for unmatched left)
FULL OUTER JOIN = everything from both sides

11. Subqueries

A subquery is a query inside another query, wrapped in parentheses.

-- Subquery in WHERE — employees earning more than average
SELECT name, salary
FROM   employees
WHERE  salary > (SELECT AVG(salary) FROM employees);
-- AVG = 75833. Returns: Alice (90k), Dave (80k), Frank (90k)

-- Subquery in FROM (derived table)
SELECT dept, avg_sal
FROM  (SELECT dept, AVG(salary) AS avg_sal
       FROM employees
       GROUP BY dept) AS dept_summary
WHERE avg_sal > 75000;

-- EXISTS — true if subquery returns at least one row
SELECT name
FROM   employees e
WHERE EXISTS (
    SELECT 1 FROM employees m
    WHERE m.emp_id = e.manager_id
);

12. ⭐ Second Highest Salary — THE Classic TCS Question

📋 Exam info
This query appears in every TCS exam, every sprint, every PRA. Learn all 3 approaches. 
-- APPROACH 1: Subquery (most commonly accepted answer)
SELECT MAX(salary) AS second_highest
FROM   employees
WHERE  salary < (SELECT MAX(salary) FROM employees);
-- MAX is 90000 → find MAX below 90000 → returns 80000 (Dave)

-- APPROACH 2: LIMIT with OFFSET (MySQL)
SELECT DISTINCT salary
FROM   employees
ORDER BY salary DESC
LIMIT 1 OFFSET 1;
-- Skip the 1st row (90000), take the next → 80000

-- APPROACH 3: Dense rank using subquery (handles ties correctly)
SELECT salary AS second_highest
FROM  (SELECT DISTINCT salary
       FROM employees
       ORDER BY salary DESC
       LIMIT 2) AS top_two
ORDER BY salary ASC
LIMIT 1;

-- nth highest salary pattern (generalised — for Nth replace 1 with N-1)
SELECT MAX(salary)
FROM   employees
WHERE  salary NOT IN (
    SELECT TOP 1 salary
    FROM employees
    ORDER BY salary DESC
);
💡 Exam tip
Always use DISTINCT when finding nth highest — otherwise duplicate salaries (Alice and Frank both earn 90k) break the result. The second highest without DISTINCT could return 90000 again.

14. UNION vs UNION ALL

-- UNION: combines results, removes duplicates
SELECT name FROM employees WHERE dept = 'IT'
UNION
SELECT name FROM employees WHERE salary > 80000;
-- Alice appears in both sets but is listed ONCE

-- UNION ALL: combines results, keeps ALL duplicates
SELECT name FROM employees WHERE dept = 'IT'
UNION ALL
SELECT name FROM employees WHERE salary > 80000;
-- Alice appears TWICE (once from each query)
🔑 Key fact
Both queries in a UNION must have the same number of columns with compatible data types. UNION ALL is faster because it skips the duplicate-check step.

15. Views

A view is a saved SELECT query — a virtual table. It doesn't store data; it runs the query every time you access it.

-- Create a view for IT department employees
CREATE VIEW it_employees AS
SELECT emp_id, name, salary
FROM   employees
WHERE  dept = 'IT';

-- Query the view like a normal table
SELECT * FROM it_employees;

-- Drop a view
DROP VIEW it_employees;
🔑 Key fact
Views provide security (hide certain columns/rows), simplify complex queries, and present a consistent interface even if underlying tables change.

16. Transactions — TCL

A transaction is a group of SQL operations treated as one unit — either ALL succeed or ALL fail (ACID).

-- Start transaction, make changes, then decide
START TRANSACTION;

UPDATE employees SET salary = salary + 5000 WHERE dept = 'IT';
DELETE FROM employees WHERE emp_id = 8;

-- SAVEPOINT: bookmark within a transaction
SAVEPOINT after_raise;

UPDATE employees SET dept = 'Archive' WHERE salary < 55000;

-- ROLLBACK to savepoint (undoes only what happened after it)
ROLLBACK TO after_raise;

-- COMMIT: makes all changes permanent
COMMIT;
🔑 Key fact
ACID properties: Atomicity — all or nothing. Consistency — DB goes from one valid state to another. Isolation — transactions don't interfere with each other. Durability — committed data survives crashes.

-- Get current date and datetime

✅ B) IT has 3 employees and HR has 2 employees — both exceed 1. Finance has only 1 employee so it is excluded by the HAVING clause. HAVING can absolutely use aggregate functions — that is its purpose.

📋 Exam scope

1. What is PL/SQL?

SQL can only query data. It can't make decisions, loop, or handle errors. PL/SQL (Procedural Language / SQL) is Oracle's extension that adds programming logic on top of SQL.

✅ Key fact
PL/SQL = SQL + procedural programming. It is Oracle-specific. Think of it as writing a small program inside the database.

2. Block Structure — The #1 Exam Topic

-- PL/SQL block structure
DECLARE
  -- Variables (OPTIONAL)
  v_name VARCHAR2(50);
BEGIN
  -- Your logic (MANDATORY)
  v_name := 'Darshan';
  DBMS_OUTPUT.PUT_LINE('Hello, ' || v_name);
EXCEPTION
  -- Error handling (OPTIONAL)
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Error occurred');
END;
/
💡 Exam tip
"Which section is mandatory?" → BEGIN. DECLARE and EXCEPTION are optional.
✅ Section order — memorize
DECLARE (optional) → BEGIN (mandatory) → EXCEPTION (optional) → END; (mandatory)
Memory: D-B-E-E — "Don't Bother Except Errors"

3. Variables & Data Types

DECLARE

  v_name     VARCHAR2(100);          -- variable-length text

  -- Anchor types — borrow type FROM a table column
  v_emp_name employees.first_name%TYPE;     -- same type as column

💡 Exam tip

5. Cursors

When SELECT returns multiple rows, use a cursor to process them one at a time.

✅ Implicit vs Explicit
Implicit: Oracle creates automatically for INSERT/UPDATE/DELETE/SELECT INTO. Check SQL%ROWCOUNT, SQL%FOUND.
Explicit: You declare it for multi-row SELECT. You control OPEN → FETCH → CLOSE.

Explicit Cursor — 4-step pattern

DECLARE
  -- Step 1: DECLARE
  CURSOR c_emp IS
    SELECT first_name, salary FROM employees WHERE dept_id = 10;
  v_name employees.first_name%TYPE;
  v_sal  employees.salary%TYPE;
BEGIN
  OPEN c_emp;           -- Step 2: OPEN
  LOOP
    FETCH c_emp INTO v_name, v_sal;  -- Step 3: FETCH
    EXIT WHEN c_emp%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE(v_name || ' earns ' || v_sal);
  END LOOP;
  CLOSE c_emp;          -- Step 4: CLOSE
END;
/

BEGIN
  FOR emp IN (SELECT first_name, salary FROM employees) LOOP
    DBMS_OUTPUT.PUT_LINE(emp.first_name || ' — ' || emp.salary);
  END LOOP;  -- auto OPEN, FETCH, CLOSE
END;

6. Procedures

CREATE OR REPLACE PROCEDURE give_raise (
  p_emp_id  IN  employees.employee_id%TYPE,
  p_amount  IN  NUMBER,
  p_new_sal OUT NUMBER
) IS
  v_old_sal NUMBER;
BEGIN
  SELECT salary INTO v_old_sal FROM employees WHERE employee_id = p_emp_id;
  p_new_sal := v_old_sal + p_amount;
  UPDATE employees SET salary = p_new_sal WHERE employee_id = p_emp_id;
END give_raise;
/
✅ Parameter modes
IN — read-only input (default). OUT — write-only output. IN OUT — both directions.

7. Functions

Like a procedure, but must return exactly one value.

CREATE OR REPLACE FUNCTION get_annual_salary (
  p_emp_id IN employees.employee_id%TYPE
) RETURN NUMBER IS
  v_monthly NUMBER;
BEGIN
  SELECT salary INTO v_monthly FROM employees WHERE employee_id = p_emp_id;
  RETURN v_monthly * 12;
END;
/
💡 Exam tip — Function vs Procedure
Function must return a value. Procedure need not.
Functions can be used in SQL SELECT. Procedures cannot.

8. Triggers

A trigger fires automatically on INSERT / UPDATE / DELETE events.

CREATE OR REPLACE TRIGGER trg_before_insert
BEFORE INSERT ON employees
FOR EACH ROW
BEGIN
  IF :NEW.salary < 0 THEN
    RAISE_APPLICATION_ERROR(-20001, 'Salary cannot be negative');
  END IF;
  :NEW.created_at := SYSDATE;
END;
/
ConceptDetail
BEFORE triggerFires before DML. Can modify :NEW values. For validation.
AFTER triggerFires after DML completes. For logging/auditing.
:NEWValues being inserted/updated. Available in INSERT/UPDATE.
:OLDValues before change. Available in UPDATE/DELETE.
FOR EACH ROWRow-level trigger. Needed to use :NEW/:OLD.
💡 Exam tip
In DELETE trigger, :NEW is NULL (no new row). In INSERT trigger, :OLD is NULL (no previous row). Only UPDATE has both.

9. Exception Handling

BEGIN
  SELECT salary INTO v_salary FROM employees WHERE employee_id = 9999;
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    DBMS_OUTPUT.PUT_LINE('No employee found.');
  WHEN TOO_MANY_ROWS THEN
    DBMS_OUTPUT.PUT_LINE('Multiple rows returned.');
  WHEN ZERO_DIVIDE THEN
    DBMS_OUTPUT.PUT_LINE('Cannot divide by zero.');
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE(SQLCODE || ' — ' || SQLERRM);

/
✅ Know these 4 predefined exceptions
NO_DATA_FOUND — SELECT INTO returned 0 rows.
TOO_MANY_ROWS — SELECT INTO returned > 1 row.
ZERO_DIVIDE — division by zero.
VALUE_ERROR — wrong type or value out of range.

SQLCODE = numeric error code. SQLERRM = error message text.

10. Packages

Groups related procedures, functions, and variables. Two parts: Specification (public interface) and Body (implementation).

    INSERT INTO employees(first_name, salary) VALUES(p_name, p_sal);
  END;
  FUNCTION get_salary(p_id NUMBER) RETURN NUMBER IS v_sal NUMBER;
  BEGIN
    SELECT salary INTO v_sal FROM employees WHERE employee_id = p_id;
    RETURN v_sal;
  END;

Exception raised when SELECT INTO returns more than one row?
  1. NO_DATA_FOUND
  2. VALUE_ERROR
  3. TOO_MANY_ROWS
  4. CURSOR_ALREADY_OPEN
✅ C) TOO_MANY_ROWS. SELECT INTO expects exactly one row.
Question 8
What does %TYPE do?
  1. Matches an entire table row
  2. Matches a specific column's data type
  3. Creates a record type
  4. Defines function return type
✅ B) %TYPE anchors to a single column's type. %ROWTYPE anchors to a whole row.
Question 9
Which loop needs EXIT/EXIT WHEN to prevent infinite loop?
  1. FOR LOOP
  2. WHILE LOOP
  3. Basic LOOP...END LOOP
  4. Cursor FOR loop
✅ C) Basic LOOP has no built-in stop. Without EXIT, it loops forever.
Question 10
How is a CONSTANT declared?

  2. c_rate CONSTANT NUMBER := 0.18;
  3. DECLARE CONSTANT c_rate = 0.18;
  4. FIXED c_rate NUMBER := 0.18;
✅ B) name CONSTANT datatype := value; Must be initialized on declaration.
Question 11
Correct sequence for explicit cursor?
  1. OPEN → DECLARE → FETCH → CLOSE
  2. DECLARE → FETCH → OPEN → CLOSE
  3. DECLARE → OPEN → FETCH → CLOSE
  4. OPEN → FETCH → DECLARE → CLOSE
✅ C) DECLARE → OPEN → FETCH → CLOSE.
Question 12
Which trigger can modify :NEW values?
  1. AFTER trigger
  2. INSTEAD OF trigger
  3. BEFORE trigger
  4. AUTONOMOUS trigger
✅ C) BEFORE trigger fires before the write, so :NEW can still be modified.

What does SQLCODE return in an exception handler?
  1. Error text message

  4. Line number of error

Package SPECIFICATION contains:
  1. Implementation code
